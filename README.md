# 🤖 AI 助理 - 日程管理中心

一个基于静态前端的日程管理网页，配合 Cursor AI 聊天窗口使用。用户在对话中告知日程安排，AI 会自动将数据分类写入对应的 JSON 文件，网页实时展示所有内容。

---

## 📁 项目结构

```
AI-Assistance/
├── index.html              # 主页面
├── css/
│   └── style.css           # 样式表
├── js/
│   └── app.js              # 前端逻辑
├── data/
│   ├── schedule.json       # 日程数据（本地，不提交）
│   ├── tasks.json          # 任务数据（本地，不提交）
│   ├── notes.json          # 备忘录数据（本地，不提交）
│   ├── reminders.json      # 提醒数据（本地，不提交）
│   ├── *.example.json      # 空模板（已提交，供他人/新环境使用）
└── README.md               # 使用说明（本文件）
```

---

## 📂 数据与 Git 说明

- **个人数据不提交**：`data/schedule.json`、`data/tasks.json`、`data/notes.json`、`data/reminders.json` 已加入 `.gitignore`，只会保存在你本机，**推送时不会带上，拉取时也不会被覆盖**。
- **你自己**：在本机维护上述四个文件，`git pull` 不会动它们，始终是你原来的数据。
- **别人 / 新克隆**：仓库里没有这四个文件时，网页会自动读取同名的 `data/*.example.json`（空模板），页面可正常使用；若需要让 AI 写入文件，可在 `data/` 下自行创建或复制：
  - `cp data/schedule.example.json data/schedule.json`
  - 其他 `tasks`、`notes`、`reminders` 同理。
- **多设备同步自己的数据**：不通过 Git，可自行用网盘、U 盘或脚本把 `data/*.json` 拷到另一台机器后再用。

## 🚀 快速启动

本项目为纯前端页面，由于需要通过 `fetch` 读取本地 JSON 文件，**不能直接双击 `index.html` 打开**，需要通过本地 HTTP 服务器运行。

### 方式一：Python（推荐）

```bash
cd g:\project\AI-Assistance
python -m http.server 8080
```

然后浏览器访问：**http://localhost:8080**

### 方式二：Node.js

```bash
npx serve .
```

### 方式三：VS Code Live Server

安装 VS Code 扩展 **Live Server**，右键 `index.html` → **Open with Live Server**。

## 📋 功能说明

### 1. 总览面板 (Dashboard)

- 显示今日日程数量、待办任务数、备忘录数、活跃提醒数
- 四个卡片区域分别预览各类数据的摘要

### 2. 日程安排 (Schedule)

按日期和时间管理事项，支持按日期筛选；支持**列表**与**日历**两种视图切换（日历按周展示）。

| 字段 | 说明 | 示例 |
|------|------|------|
| `id` | 唯一标识 | `sch-001` |
| `date` | 日期 | `2026-03-07` |
| `time` | 开始时间 | `09:00` |
| `endTime` | 结束时间 | `10:00` |
| `title` | 事项名称 | `团队周会` |
| `description` | 详情描述 | `讨论本周进展` |
| `category` | 类别 | `work` / `personal` / `meeting` / `health` / `study` / `other` |
| `status` | 状态 | `pending` / `in-progress` / `completed` |

### 3. 待办任务 (Tasks)

任务清单管理，支持按状态和优先级筛选。

| 字段 | 说明 | 示例 |
|------|------|------|
| `id` | 唯一标识 | `task-001` |
| `title` | 任务名称 | `完成项目报告` |
| `description` | 描述 | `整理Q1数据并生成报告` |
| `priority` | 优先级 | `high` / `medium` / `low` |
| `status` | 状态 | `pending` / `in-progress` / `completed` |
| `deadline` | 截止日期 | `2026-03-10` |
| `createdAt` | 创建日期 | `2026-03-07` |
| `tags` | 标签数组 | `["工作", "报告"]` |

### 4. 备忘录 (Notes)

以卡片形式展示笔记和重要信息。

| 字段 | 说明 | 示例 |
|------|------|------|
| `id` | 唯一标识 | `note-001` |
| `title` | 标题 | `会议纪要` |
| `content` | 正文内容 | `1. 确认方案...` |
| `createdAt` | 创建日期 | `2026-03-07` |
| `tags` | 标签数组 | `["会议"]` |

### 5. 提醒事项 (Reminders)

定时提醒管理，支持重复设置。重复类提醒会显示**下次时间**（如「下次：今天 16:00」「下次：每周一 09:00」）。

| 字段 | 说明 | 示例 |
|------|------|------|
| `id` | 唯一标识 | `rem-001` |
| `title` | 提醒内容 | `提交周报` |
| `description` | 详细说明 | `发送至邮箱` |
| `datetime` | 提醒时间 | `2026-03-08T17:00:00` |
| `repeat` | 重复方式 | `daily` / `weekly` / `monthly` / `yearly` / `once` / `none` |
| `status` | 状态 | `active` / `paused` |

## 💬 如何使用（与 AI 对话）butoul

在 Cursor 聊天窗口中，直接用自然语言告诉 AI 你的安排，例如：

### 添加日程

> "明天上午10点到11点有个产品评审会议"

AI 会将对应数据写入 `data/schedule.json`。

### 添加任务

> "帮我记一个任务：下周五之前完成季度报告，优先级高"

AI 会将对应数据写入 `data/tasks.json`。

### 添加备忘

> "记一下：服务器密码已更新为 xxx，运维联系人张三 138xxxx"

AI 会将对应数据写入 `data/notes.json`。

### 添加提醒

> "提醒我每周五下午5点提交周报"

AI 会将对应数据写入 `data/reminders.json`。

### 修改/删除

> "把明天的产品评审会议改到下午2点"
> "删除'完成季度报告'这个任务"
> "把买牛奶的任务标记为已完成"

## 🔄 数据刷新

- 页面每 **30 秒** 自动刷新数据
- 点击右上角 **🔄 刷新** 按钮可手动刷新
- AI 更新数据文件后，刷新页面即可看到最新内容

## 🎨 界面特性

- **响应式设计**：适配桌面和移动设备
- **侧栏导航**：可折叠的侧栏，快速切换各面板
- **实时时钟**：侧栏底部显示当前时间和日期
- **全局搜索**：顶部搜索框可同时搜索日程、任务、备忘录、提醒，结果按类型分组展示
- **日程快捷筛选**：今天 / 本周 / 下周 / 全部一键筛选；支持**列表**与**日历（按周）**视图切换
- **提醒下次时间**：每日/每周/每月重复的提醒会显示「下次：今天 16:00」等文案
- **筛选功能**：日程按日期筛选，任务按状态/优先级筛选
- **状态徽章**：不同状态和优先级以彩色徽章区分
- **卡片视图**：备忘录以卡片形式展示，清晰直观
- **深色侧栏 + 浅色内容区**：现代化对比配色
- **网页快速添加**：各面板提供「+ 添加」按钮，可在页面直接添加日程/任务/备忘/提醒，数据保存在浏览器本地（localStorage），与 AI 写入的 JSON 合并展示
- **系统提醒**：支持浏览器通知权限，到点对「活跃」的重复提醒进行弹窗提醒（每 30 秒检测一次）
